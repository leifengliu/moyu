# 部署说明

## 1. 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | Java 运行环境 |
| Maven | 3.9+ | 项目构建 |
| MySQL | 8.0+ | 数据库 |
| Redis | 7.x | 缓存服务 |
| Nginx | 1.24+ | 反向代理(可选) |

## 2. 数据库初始化

```bash
# 1. 连接MySQL
mysql -u root -p123456

# 2. 执行初始化脚本
source moyu-server/sql/init.sql;

# 3. 验证
USE moyu_coffee;
SHOW TABLES;
SELECT COUNT(*) FROM prd_product;  -- 应为8
```

## 3. 后端启动

### 3.1 修改配置
编辑 `moyu-server/src/main/resources/application.yml`:
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/moyu_coffee?...  # 确认连接信息
    username: root
    password: 123456
  data:
    redis:
      host: localhost      # Redis地址
      port: 6379           # Redis端口

wx:
  miniapp:
    appid: your-appid      # 替换为实际AppID
    secret: your-secret    # 替换为实际Secret
```

### 3.2 编译启动
```bash
cd moyu-server
mvn clean package -DskipTests
java -jar target/moyu-server-1.0.0-SNAPSHOT.jar
```

### 3.3 验证
- API文档: http://localhost:8080/doc.html
- 健康检查: http://localhost:8080/actuator/health (如果启用)

## 5. 接口调试流程

使用 Knife4j 文档页面 (http://localhost:8080/doc.html):

1. **登录**: POST `/api/v1/user/login/wechat`
   ```json
   { "code": "test_code", "nickname": "测试用户" }
   ```
   → 获取 token

2. **设置全局Token**: 在 Knife4j 页面右上角 "Authorize" 填入 `Bearer {token}`

3. **测试接口**:
   - 商品: GET `/api/v1/product/categories`
   - 加购: POST `/api/v1/cart/add`
   - 下单: POST `/api/v1/order/submit`
   - 支付: POST `/api/v1/order/{id}/pay`

## 6. 小程序前端部署

### 6.1 开发环境
1. 下载 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 导入项目目录 `moyu-miniapp`
3. 修改 `utils/api.js` 中的 `BASE_URL` 为后端服务地址
4. 点击编译预览

### 6.2 生产环境
1. 在微信公众平台注册小程序
2. 配置服务器域名 (request合法域名)
3. 上传代码 → 提交审核 → 发布

## 7. 生产环境建议

### 7.1 性能优化
- MySQL: 开启查询缓存, 配置连接池(推荐 HikariCP, Spring Boot默认)
- Redis: 开启持久化, 配置最大内存
- Nginx: 开启 gzip, 配置静态资源缓存

### 7.2 安全加固
```yaml
# 生产环境配置
jwt:
  secret: ${JWT_SECRET}  # 环境变量, 不使用硬编码

wx:
  miniapp:
    appid: ${WX_APPID}
    secret: ${WX_SECRET}

spring:
  datasource:
    password: ${DB_PASSWORD}
```

### 7.3 监控
- 应用监控: Spring Boot Actuator
- 日志收集: ELK / 阿里云SLS
- APM: SkyWalking / 阿里云ARMS

### 7.4 高可用
- 数据库: 主从复制 + 读写分离
- Redis: 哨兵模式 / Cluster
- 应用: 多实例 + Nginx负载均衡

## 8. 常见问题

### Q: 数据库连接失败
检查 MySQL 服务是否启动, 账号密码是否正确, 数据库是否已创建。

### Q: Redis 连接失败
检查 Redis 服务是否启动, 端口是否被占用, 密码是否正确。

### Q: 微信登录失败
检查 `appid` 和 `secret` 是否正确, 小程序是否已配置服务器域名。

### Q: Token 失效
Token 有效期 7天, 过期后需重新登录。可在 Filter 中增加自动续期逻辑。
