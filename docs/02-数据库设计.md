# 数据库设计文档

## 1. 设计规范

- 数据库名: `moyu_coffee`
- 字符集: `utf8mb4` / `utf8mb4_unicode_ci`
- 引擎: InnoDB
- 主键: BIGINT AUTO_INCREMENT
- 金额字段: DECIMAL(10,2)
- 时间字段: DATETIME
- 索引命名: `uk_` 唯一索引, `idx_` 普通索引

## 2. ER 图关系

```
sys_user ──1:N── crt_cart_item
sys_user ──1:N── ord_order
sys_user ──1:1── wlt_account
sys_user ──1:1── pts_account
sys_user ──1:N── cpn_user_coupon
sys_user ──1:N── wlt_transaction
sys_user ──1:N── pts_record

prd_category ──1:N── prd_product
prd_product ──1:N── prd_product_spec

ord_order ──1:N── ord_order_item
ord_order ──1:N── ord_order_log
ord_order ──N:1── cfg_store
ord_order ──N:1── cpn_user_coupon (优惠券)
```

## 4. 索引策略

| 表 | 索引 | 类型 | 用途 |
|-----|------|------|------|
| sys_user | uk_openid | UNIQUE | 微信登录查用户 |
| sys_user | uk_phone | UNIQUE | 手机号登录 |
| prd_category | uk_code | UNIQUE | 编码查询 |
| crt_cart_item | uk_user_product_spec | UNIQUE | 购物车去重 |
| crt_cart_item | idx_user | NORMAL | 用户购物车列表 |
| ord_order | uk_order_no | UNIQUE | 订单号查询 |
| ord_order | idx_user_id | NORMAL | 用户订单列表 |
| ord_order | idx_status | NORMAL | 按状态筛选 |
| wlt_transaction | idx_user | NORMAL | 用户交易记录 |
| pts_record | idx_user | NORMAL | 用户积分记录 |
| cpn_user_coupon | idx_user | NORMAL | 用户优惠券 |
| cpn_user_coupon | idx_status | NORMAL | 按状态筛选 |

## 5. 种子数据说明

初始化脚本 (`moyu-server/sql/init.sql`) 包含以下种子数据:

- **4个商品分类**: 新品、经典咖啡、特调、周边
- **8款商品**: 拿铁、美式、冰咖啡、卡布奇诺、摩卡、抹茶拿铁、可颂、咖啡豆
- **8×4=352条规格数据**: 每个咖啡类商品有 SIZE(3)+TEMP(4)+SUGAR(4) 规格
- **4个储值套餐**: 100/200/500/1000元
- **4个积分商品**: 拿铁券、美式券、可颂券、咖啡豆
- **3张优惠券**: 新人8折券、满50减10、任意消费减5
- **1个测试用户**: openid=test_openid_001, 余额128元, 积分360
- **3条测试订单**: 2已完成 + 1待取餐
- **交易/积分/优惠券** 对应记录数据
