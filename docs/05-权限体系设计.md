# 权限体系设计

## 1. 认证架构

```
客户端请求
    │
    ▼
JwtAuthenticationFilter (OncePerRequestFilter)
    │
    ├── 从 Authorization header 提取 Bearer token
    ├── JwtTokenProvider.validateToken() 校验签名+过期
    ├── 解析 userId, openid
    └── UserContext.setCurrentUser(userId, openid)
    │
    ▼
Controller / Service
    │
    ├── UserContext.getCurrentUserId() 获取当前用户
    │    └── 未登录抛 BusinessException(401)
    │
    └── 业务逻辑...
    │
    ▼
finally: UserContext.clear() (ThreadLocal清理)
```

## 2. JWT Token 设计

### 2.1 Token 生成
```java
Jwts.builder()
    .subject(String.valueOf(userId))
    .claim("openid", openid)
    .issuedAt(now)
    .expiration(now + 7天)
    .signWith(HMAC-SHA256 secret key)
    .compact()
```

### 2.2 Token 内容
```json
{
  "sub": "1",
  "openid": "test_openid_001",
  "iat": 1717171200,
  "exp": 1717776000
}
```

### 2.3 配置
```yaml
jwt:
  secret: moyu-coffee-jwt-secret-key-2026-must-be-at-least-256-bits-long
  expiration: 604800000  # 7天
```

## 5. 安全措施

### 5.1 防重放
- Token 有效期 7天
- 敏感操作(支付、退款) 需二次确认

### 5.2 数据隔离
- 所有用户数据查询均添加 `userId` 条件过滤
- Controller 层通过 `UserContext.getCurrentUserId()` 获取当前用户
- 禁止通过参数传递 userId (防止越权)

```java
// 正确: 从Token获取userId
Long userId = UserContext.getCurrentUserId();
orderMapper.selectList(wrapper.eq(Order::getUserId, userId));

// 错误: 从请求参数获取userId (可被篡改)
orderMapper.selectList(wrapper.eq(Order::getUserId, request.getUserId()));
```

### 5.3 并发安全
- 储值扣减: `UPDATE wlt_account SET balance = balance - ? WHERE user_id = ? AND balance >= ?` (乐观锁)
- 积分扣减: `UPDATE pts_account SET balance = balance - ? WHERE user_id = ? AND balance >= ?`
- 库存扣减: `UPDATE pts_product SET stock = stock - 1 WHERE id = ? AND stock > 0`
- 订单操作: `@Transactional` 保证事务一致性

### 5.4 验证码安全
- 短信验证码: Redis存储, 5分钟过期
- 60秒内禁止重复发送
- 验证成功后立即删除

### 5.5 敏感信息
- 手机号: 存储加密/脱敏显示(138****8000)
- JWT Secret: 配置文件独立管理, 生产环境使用环境变量
